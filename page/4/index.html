1.如何高效的切圆角？

切圆角共有三种方案：
- CornerRadius + MasksToBounds：适用于单个视图或视图不在列表上且量级较小的情况，会触发离屏渲染
- CAShapeLayer + UIBezierPath：会触发离屏渲染，性能消耗严重，不推荐使用
- Core Graphics：推荐使用（不会触发离屏渲染）

2.什么是隐式动画和显示动画？

- 隐式动画指的是改变属性值而产生的默认的过度动画（如backgound、cornerRadius等），不需要初始化任何类，系统自己处理动画属性；
- 显示动画是指自己创建一个动画对象并附加到layer上，如 CAAnimation、CABasicAnimation、CAKeyframeAnimation。

3.UIView 和 CALayer 的区别？

- UIView 是负责事件的传递，参与视图响应链
- CALayer 负责视图的显示功能

【iOS】iOS性能优化（如何优化APP的电量）

发表于 2016-05-26 更新于 2020-03-29 分类于 iOS ， 性能优化

如何优化App的电量？

程序的耗电方面主要在一下四个方面：

  • Cpu处理
  • 定位
  • 网络
  • 图像

优化的途径主要提现在以下的几个方面：

  • 尽量降低CPU、GPU的消耗
  • 尽量少用定时器
  • 优化I/O操作
    • 不要频繁写入小数据，而是等到积攒一定的数量再写入
    • 读写大量的数据可以使用 Dispatch_io, GCD内部已经做了优化。
    • 数据量比较大时，建议使用数据库
  • 网络方面的优化
    • 减少压缩网络数据（XML-> JSON -> ProtoBuf）,如果可能建议使用ProtoBuf
    • 如果请求的返回数据相同，可以使用 NSCache 进行缓存
    • 使用断点续传，避免因网络失败后要重新下载
    • 网络不可用的时候，不要尝试进行网络请求
    • 长时间的网络请求，要提供可以取消的操作
    • 采取批量传输。下载视频流的时候，尽量一大块一大块的进行下载，广告可以一次下载多个
  • 定位层面的优化
    • 如果只是需要快速确定用户位置，最好用 CLLocationManager 的 requestLocation 方法，定位完成后，会自动让定位硬件断电
    • 如果不是导航应用，尽量不要实时更新位置，定位完毕就关掉定位服务
    • 尽量降低定位精度，比如尽量不要使用精度最高的 kCLLocationAccuracyBest
    • 需要后台定位时，尽量设置 pausesLocationUpdatesAutomatically 为 YES，如果用户不太可能移动的时候，系统会自动暂停位置更新
    • 尽量不要使用 startMonitoringSignificantLocationChanges，有限考虑 startMonitoringForRegion
  • 硬件检测优化
    • 用户移动、摇晃、倾斜设备时，会产生动作（motion）事件，这些时间由加速计 、陀螺仪，磁力计等硬件检测。在不需要检测的场合，应该及时关闭这些硬件。
